# Resume Tailor — AGENTS.md

## Quick start

```bash
# Backend (resume_tailor_project/python_backend/)
echo "GROQ_API_KEY=..." > .env
source venv/bin/activate
python server.py                              # Port from PORT env (default 5000)
# Extension + tests hardcode port 5001 — set PORT=5001 or test will fail

# LaTeX generator (Latex_from_Json Engine/)
python generate.py data.json                  # build (gen + compile)
python generate.py data.json --no-pdf         # .tex only, skip PDF
python generate.py data.json --no-fit         # skip 1-page fitting
python generate.py generate data.json         # explicit generate subcommand
```

## Architecture

- **`resume_tailor_project/python_backend/server.py`** — Flask server (port 5000, but tests/extension use 5001). POST `/generate` calls Groq API (llama-3.3-70b-versatile, via OpenAI client with `base_url="https://api.groq.com/openai/v1"`), returns JSON + base64 PDF/TEX.
- **`resume_tailor_project/browser_extension/`** — Chrome MV3 extension. `popup.js` uses `chrome.scripting.executeScript` to get highlighted text (content.js is intentionally empty).
- **`Latex_from_Json Engine/generate.py`** — Standalone LaTeX generator (JSON → .tex → PDF). Three subcommands: `generate`, `compile`, `build` (implicit default when no subcommand given). Auto-fits to 1 page through 7 levels of tightening. Supports `tectonic` (default) and `pdflatex`.
- **`resume_tailor_project/templates/prompt_template.txt`** — LLM prompt that uses `{{Insert Here}}` for JD and `{company_name}` for company name. The LLM returns JSON matching `data.json` schema.
- **`resume_tailor_project/json_latex_pdf_engine/`** — Empty/unused. The real engine is `Latex_from_Json Engine/`.

## Tests

```bash
# API tests (server must be running on port 5001)
python3 -m pytest tests/test_api.py -v --tb=short 2>&1 | tee tests/api_report.txt

# Load test (requires k6, server on port 5001)
k6 run tests/load_test.js 2>&1 | tee tests/k6_report.txt
```

## Key quirks

- **No `requirements.txt`** — deps are in `server.py` imports: flask, flask-cors, flask-limiter, openai, python-dotenv, pypdf.
- **Port mismatch** — `server.py` defaults to `PORT` env (5000), but tests, extension, and load test all hardcode `127.0.0.1:5001`. Set `PORT=5001` or change the hardcoded values.
- **Rate limiting** — default 10 requests/minute per IP (configurable via `RATE_LIMIT` env, stored in-memory).
- **Auth** — optional `AUTH_TOKEN` env var; when set, `Authorization: Bearer <token>` is required.
- **LaTeX engine resolution** — prefers tectonic, falls back to pdflatex, returns None if neither found.
- **Input validation** — company name must be <=100 chars, alphanumeric/dash/dot/underscore only; JD max 15k chars; body max ~5MB.
- **Generated data** — `data_*.json` files are gitignored and auto-generated by `/generate` endpoint.
- **Template placeholders** — `{{FONT_SIZE}}`, `{{ITEM_SEP}}`, `{{MARGIN_LINES}}` are auto-injected by the fitting engine (DO NOT put in JSON). Simple placeholders like `{{NAME}}` come from JSON. Structured keys (`experience`, `projects`, `skills`, `achievements`) are auto-built into LaTeX blocks.
- **No CI/CD, no package.json, no formatter/linter config** — just raw Python + JS.
